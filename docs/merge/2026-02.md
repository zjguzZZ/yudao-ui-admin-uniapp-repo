## `b9ef57a5` 统一提取包

不要拆到多个批次。建议一个分支内处理，按功能点提交。

### token 响应式

需要提取：

- `src/store/token.ts`
  - 增加 `nowTime = ref(Date.now())`。
  - 增加 `updateNowTime()`，内部更新 `nowTime.value`，并返回 `useTokenStore()`。
  - `isTokenExpired` 使用 `nowTime.value`，不要继续直接用 `Date.now()`。
  - 不要把上游对 `isRefreshTokenExpired` 的 `nowTime.value` / `refreshTokenExpireTime` 改动照搬过来。
  - 在 `setTokenInfo()` 开头调用 `updateNowTime()`。
  - 在 `refreshToken()` 的 `finally` 中调用 `updateNowTime()`。
  - 在 `logout()` 的 `finally` 中调用 `updateNowTime()`。
  - 在 `tryGetValidToken()` 开头调用 `updateNowTime()`。
  - 返回对象中暴露 `updateNowTime`。
- `src/http/interceptor.ts`
  - 取请求头 token 时，从 `tokenStore.validToken` 改为 `tokenStore.updateNowTime().validToken`。

本轮额外适配当前项目的直接 token 读取点：

- `src/http/http.ts`
  - 修复并发 401 场景下，部分请求在 refresh 成功前已经 reject 的问题。
  - 请求队列同时保存 `resolve`、`reject` 和原始请求参数；refresh 成功后统一重试，refresh 失败后统一 reject 并登出。
  - 增加 `refreshFailedReason`，避免 refresh 失败处理期间新来的 401 请求重新入队后挂起。
- `src/api/infra/file/index.ts`
  - 上传文件时读取 `tokenStore.updateNowTime().validToken`。
- `src/pages-infra/web-socket/index.vue`
  - WebSocket URL 在没有 refresh token 时读取 `tokenStore.updateNowTime().validToken`。
- `src/App.vue`
  - `onShow` 补偿加载字典前刷新登录态计算。
- `src/router/interceptor.ts`
  - 路由登录判断前刷新登录态计算。
- `src/tabbar/store.ts`
  - tabbar 切换权限判断前刷新登录态计算。

明确决策：refresh token 过期逻辑不从上游照搬。

- 当前后端不返回 refresh token 过期时间，所以不要恢复 `refreshTokenExpireTime` 本地存储。
- 双 token 模式下，`isRefreshTokenExpired` 继续按当前项目逻辑返回 `false`。
- refresh token 最终以刷新接口失败或 401 作为失效依据。
- `nowTime` 只用于 access token 过期判断。

## 验证要求

- 单 token：登录成功后请求带 token；手动把 `accessTokenExpireTime` 改成过去时间后，`validToken` 应为空。
- 双 token：手动把 `accessTokenExpireTime` 改成过去时间后，请求触发 refresh；刷新成功后重放请求；刷新失败后清理登录态并跳转登录页。
- 双 token 并发：多个业务请求同时遇到 access token 过期时，只触发一次 refresh；所有请求都应在 refresh 成功后重试成功。
- `SKIP_OPEN_DEVTOOLS=true pnpm build:mp:prod` 不自动打开开发者工具。
- `pnpm type-check`
